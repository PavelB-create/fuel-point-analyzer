# Техническое задание на разработку проекта "FuelPoint Analyzer"

## 1. Концепция и название проекта
**Название:** Аналитическая система мониторинга топливной эффективности для частных автовладельцев с целью оптимизации эксплуатационных расходов.

**Описание:** Веб-сервис позволяет водителям вести точный учет заправок по методу «полного бака». В отличие от штатных систем автомобиля, сервис анализирует данные в долгосрочной перспективе, выявляет реальный расход топлива в зависимости от бренда АЗС и типа топлива, а также визуализирует географию перемещений.

---

## 2. Целевая аудитория и роли
Сервис ориентирован на автовладельцев, которые ведут учет расходов и хотят выявить наиболее качественные сети АЗС для своего автомобиля.

**Роли пользователей:**
*   **Гость:** Просмотр презентационной страницы, регистрация и вход.
*   **Авторизованный пользователь:** 
    *   Управление списком собственных автомобилей (Гараж).
    *   Добавление записей о заправках через интерактивную карту.
    *   Просмотр персональной аналитики и графиков расхода.
*   **Администратор:** Управление справочниками сетей АЗС и модерация данных.

---

## 3. Схема данных (Сущности)
В системе реализованы 3 взаимосвязанные кастомные модели:

1.  **Vehicle (Автомобиль):**
    *   `owner` (Связь с User) — владелец авто.
    *   `brand`, `model` — идентификация ТС.
2.  **FuelNetwork (Сеть АЗС):**
    *   `name` — бренд заправки (Лукойл, Газпром и т.д.).
    *   `price_92`, `price_95`, `price_diesel` — актуальные цены, обновляемые через внутренний сервис интеграции.
3.  **Refueling (Запись о заправке):**
    *   `vehicle` (ForeignKey) — к какому авто относится.
    *   `network` (ForeignKey) — на какой АЗС была заправка.
    *   `odometer` — пробег на момент заправки.
    *   `fuel_amount` — объем залитого топлива.
    *   `fuel_type` — выбранный тип топлива.
    *   `price_total` — итоговая стоимость (рассчитывается автоматически).
    *   `latitude`, `longitude` — геопозиция АЗС.

---

## 4. Основные сценарии пользователя (User Stories)
1.  **Регистрация и создание авто:** Пользователь создает аккаунт, добавляет автомобиль (марка, модель).
2.  **Добавление заправки через карту:** Пользователь открывает форму, ищет АЗС через поиск 2ГИС или кликает на карту. Система автоматически подставляет координаты и актуальную цену за литр для выбранного бренда.
3.  **Автоматический расчет:** При вводе количества литров JavaScript мгновенно вычисляет общую стоимость заправки.
4.  **Анализ эффективности:** На главном дашборде пользователь видит средний расход своего авто и рейтинг брендов АЗС (где расход был минимальным).
5.  **Визуализация трендов:** Пользователь анализирует график изменения расхода, чтобы вовремя заметить технические неисправности (резкий рост расхода).

---

## 5. Технический стек и интеграции
*   **Backend:** Django 5.x (ORM, Auth, Templates).
*   **Data Science:** 
    *   `Pandas` — группировка данных по сетям АЗС, расчет скользящего среднего расхода.
    *   `Matplotlib` — генерация аналитических графиков.
*   **Внешние API:**
    *   `2GIS MapGL API` — визуализация карты и работа с маркерами.
    *   `2GIS Places API` — реализация поиска организаций (АЗС) по геоданным.
*   **Безопасность:** `python-dotenv` для изоляции API-ключей.
